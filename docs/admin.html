Requirement levels
A schema may have a number of requirement levels.
These should be defined in the schema configuration as an ordered, comma separated list, starting with least strict.
Example: MAY,SHOULD,MUST
In the schema, these requirement levels are specified before each rule, enclosed in backticks.
Example: `MUST` foaf:givenName xsd:string
NOTE: Appending  NOT to a requirement level is ignored by the UI. Negation should be added using standard syntax.
Examples: `SHOULD NOT` !foaf:givenName xsd:string
